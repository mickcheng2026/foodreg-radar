# 食規雷達 — 部署上線步驟（給非技術使用者）

> 全程預計 **15–20 分鐘** 完成。完成後您和朋友只要打開一個網址就能看到每天自動更新的食品法規資訊。

---

## 你會得到什麼

- 一個公開網址（例如 `https://你的帳號.github.io/foodreg-radar/`）
- 每天台灣時間早上 9:00 自動抓取最新公告
- **完全免費** — 沒有任何月費

---

## 步驟 1：註冊 GitHub 帳號（如果還沒）

1. 打開 https://github.com/signup
2. 輸入 Email、密碼、帳號名稱（這個帳號名稱會出現在網址裡，建議短一點，例如 `mick123`）
3. 完成驗證

> 💡 如果已經有 GitHub 帳號，直接跳到步驟 2。

---

## 步驟 2：建立新的程式碼倉庫（Repository）

1. 登入 GitHub 後，點擊右上角 **「+」** → **「New repository」**
2. 填入以下資訊：
   - **Repository name**：`foodreg-radar`（這會變成網址的一部分）
   - **Description**：`食規雷達 — 食品法規最新動態`（可選）
   - 選 **Public**（公開，這樣 GitHub Pages 才能免費用）
   - **不要**勾選「Add a README file」、「Add .gitignore」、「Choose a license」
3. 點底部綠色按鈕 **「Create repository」**

> 完成後您會看到一個空 repo 的頁面，有兩個區塊指示「Quick setup」。

---

## 步驟 3：上傳專案檔案

最簡單的方式：**用 GitHub 網頁直接拖拉上傳**。

1. 在剛建立的 repo 頁面，找到藍色字 **「uploading an existing file」** 的連結，點下去
   （或直接進 `https://github.com/你的帳號/foodreg-radar/upload/main`）

2. 開啟 Finder，找到這個資料夾：
   ```
   /Users/mick/Desktop/claude code files/teamnotes-vault/條文自動更新 網站/
   ```

3. **全選**該資料夾裡所有的檔案與子資料夾（⌘+A），**整批拖拉**到 GitHub 上傳區
   - 包含：`index.html`、`data/`、`scripts/`、`.github/`、`.gitignore`、`部署步驟.md` 等
   - **不要上傳**：`prototype.html`（這只是給您看的草稿）

4. 等所有檔案出現在頁面上（可能要 30 秒到 1 分鐘）

5. 拉到頁面底部，在「Commit changes」框：
   - 上方輸入：`首次上傳食規雷達專案`
   - 點綠色按鈕 **「Commit changes」**

---

## 步驟 4：啟用 GitHub Pages（讓網站可以被開啟）

1. 在您的 repo 頁面上方，點 **「Settings」**（齒輪圖示）
2. 左側選單找到 **「Pages」**，點進去
3. 在 **「Build and deployment」** 區塊：
   - **Source** 選 **「Deploy from a branch」**
   - **Branch** 選 **「main」**，資料夾選 **「/ (root)」**
   - 點 **「Save」**
4. 等待約 1–3 分鐘
5. 重新整理該頁，最上方會出現綠色框：
   ```
   ✓ Your site is live at https://你的帳號.github.io/foodreg-radar/
   ```
6. **點那個網址** → 看到網站正常顯示，恭喜上線了！

---

## 步驟 5：確認自動更新功能

1. 回到 repo 主頁面
2. 點上方 **「Actions」** 分頁
3. 第一次進入會看到提示「Workflows aren't being run on this forked repository」之類的：
   - 點 **「I understand my workflows, go ahead and enable them」**
4. 左側會出現「每日自動更新食規資料」這個 workflow
5. 點它，然後點右上角 **「Run workflow」** → 選 `main` → **「Run workflow」**
6. 等 2–3 分鐘，工作完成（綠色勾勾）就代表自動更新沒問題

> 之後每天台灣時間早上 9:00 都會自動跑。

---

把這個網址傳給對方就好：
```
https://你的帳號.github.io/foodreg-radar/
```

對方只要打開就能看到最新資訊，不用安裝任何東西、不用註冊。

## 常見問題

### ❓ 我可以換網站名稱嗎？
可以。打開 `index.html`，搜尋「食規雷達」並全部替換成您想要的名字。

### ❓ 想加 AI 摘要怎麼辦？
您之前選了零元方案不加 AI。未來想加，告訴我，我會幫您：
1. 申請 Anthropic API key（預付 $5 美金可用很久）
2. 修改爬蟲程式，每筆新進資料自動產生中文重點

### ❓ ISO / Codex / US FDA 為什麼資料比較少？
這幾個國際來源的網站結構比較複雜，目前的爬蟲只能抓到部分資訊。
食藥署是主要來源（每天約 10 筆新公告），這是衛生管理員最常用的。
其他來源未來可以再優化。

### ❓ 想增加新的資料來源（例如 CAS、農業部）？
告訴我網址，我幫您寫新的爬蟲模組。

### ❓ 衛生管理員想收 Email 通知怎麼辦？
未來可加上：
- RSS（讓對方用 Feedly / Inoreader 訂閱）
- LINE Notify 推播
- Email 訂閱

## 技術細節（可略過）

- **架構**：純靜態 HTML/JS + Python 爬蟲 + GitHub Actions cron
- **資料**：每筆儲存在 `data/items.json`
- **去重**：以 URL 為主鍵，新版本覆蓋舊版本但保留 `first_seen` 時間
- **排程**：`.github/workflows/update-data.yml`（cron `0 1 * * *` UTC = 09:00 台灣）
- **修改週期**：想改為每 6 小時一次，把 cron 改成 `0 */6 * * *`
