## 🎯 最常用的 3 種情境

### A. 跟 Claude Code 對話（最輕鬆）

直接在對話框打：

| 您要做的事 | 對 Claude Code 說 |
|---|---|
| 把最新公告抓下來、產 AI 重點、上線 | **「FoodReg 抓新資料推送」** |

| 改網站名稱、配色等 | **「FoodReg 把標題改成 ___」** |
| 增加新資料來源 | **「FoodReg 加 ___ 來源」**（CAS、農業部等）|

我會自動做完上述所有事 + 推上 GitHub，您 1-2 分鐘後重新整理網站就會更新。

### B. 自己跑終端機（不用 Claude Code 也行）

打開**終端機 (Terminal)**，輸入：

```bash
cd "/Users/mick/Desktop/claude code files/teamnotes-vault/條文自動更新 網站"

# 推送現有本機變更
./推送.sh

# 先抓最新公告再推送（會跑爬蟲約 60-90 秒）
./推送.sh --refresh
```

### C. 完全不管它（讓系統自動跑）

什麼都不做。GitHub Actions **週一到週五 台灣時間早上 7:50** 會自動：
1. 跑爬蟲抓食藥署、EFSA、各國食安事件等最新資料
2. 套用 AI 重點（精選＋規則式自動產生）
3. **自動把外文標題翻成中文**（免費的 Google 翻譯）
4. 更新 `data/items.json`、自動 commit 推回 GitHub
5. 網站幾分鐘內更新

✅ 排程已經會「自動產生 AI 重點 + 中文翻譯」，多數情況**完全不用手動**。
ℹ️ 唯一沒做的是「逐篇用 Claude 重新精讀的高品質摘要」（那需要 API 金鑰、要花錢）。想要某幾筆更精緻的摘要時，再用 A 方式請我處理即可。

## 📊 怎麼確認網站有更新

1. 打開 https://mickcheng2026.github.io/foodreg-radar/
2. **強制重新整理**：⌘ + Shift + R（繞過快取）
3. 確認三件事：
   - 頂部「最後更新」時間是不是今天
   - 卡片列表第一張的日期是不是最新
   - 有 AI 摘要的卡片是否顯示綠色「AI 重點」標籤

## 🤖 怎麼確認自動排程有跑

1. 打開 https://github.com/mickcheng2026/foodreg-radar/actions
2. 看左側「每日自動更新食規資料」
3. 點進去 → 看最近一次執行
   - 綠色勾 ✅ = 正常
   - 紅色叉 ❌ = 有錯，可以點進去看 log

## ⚠️ 常見問題

### Q1：網站打開卡在「載入資料中」永遠不動
**原因**：可能是 GitHub Pages 還在部署，或瀏覽器快取舊版
**解法**：等 3 分鐘後，⌘ + Shift + R 強制重新整理

### Q2：推送顯示 `Updates were rejected`
**原因**：線上有改動本機沒同步
**解法**：終端機跑 `git pull origin main` 再跑 `./推送.sh`

### Q3：推送顯示 `refusing to allow ... without workflow scope`
**原因**：PAT 沒勾 `workflow` 權限
**解法**：去 https://github.com/settings/tokens → 點 `foodreg` → 勾 `workflow` → Update token → 重跑 `./推送.sh`

### Q4：推送顯示 `Authentication failed`
**原因**：PAT 過期或被撤銷
**解法**：重做一次 [設定git推送.md](./設定git推送.md) 的步驟 1（建新 PAT）；然後跑 `./推送.sh` 重新輸入

### Q5：GitHub Actions 紅色叉，網站好幾天沒更新
**原因**：可能是食藥署 / EFSA 網站結構改了，爬蟲失敗
**解法**：跟 Claude Code 說「**FoodReg 爬蟲壞了**」，我會去看 log + 修

### Q6：想加新的資料來源（CAS、農業部、各縣市衛生局…）
跟 Claude Code 說「**FoodReg 加 CAS 來源，網址：xxx**」

### Q7：想改網站名稱、Logo、配色
跟 Claude Code 說「**FoodReg 把名稱改成 ___**」、「**配色改成深色系**」等

### Q8：衛生管理員回饋想加某功能（按收藏、Line 通知…）
告訴我，多數功能 30 分鐘內可加上。
